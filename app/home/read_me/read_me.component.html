API Management/Monitoring

If you scroll down you will see the list of error codes and the data dictionary.

Did you know? (no, i don't care too)
Parkninja provides ReSTful web services for android and ios clients. One of the main concepts of ReST is to make use of Http methods to access logical resources. For parkninja we only use GET and POST to make things simpler.
  • GET - Read data / Requests that will not affect state
  • POST - Update, Delete, Create / Requests that affect state
We use plural nouns for the resources.
  • /api/user -> /api/users (v1 is pretty bad sorry peace yo)
Accessing sub-resources
  • /api/v2/parking/ -> Read all parking lots data
  • /api/v2/parking/1 -> Read parking lot data with id = 1
Query parameters for sorting, paging etc. -> this is not implemented yet but we will when motivation hits us in the face :)
  • /api/v2/parking?limit=1&sorting=asc
Http status codes
  • HttpStatus 200 -> Successful requests
  • HttpStatus 400 -> Something is wrong with the JSON parameters that the client sent.
  • HttpStatus 500 -> Internal Server Error, sounds pretty fckng bad. This should not happen contact the noob API devs immediately.
Authentication -> We are using Oauth2 for generating access tokens because you know it's cool. Since we own the apps that will access our API's we just provide them with a non-expiring access token because we trust them. In case some third party apps wants to access our API's we will provide another way of providing access tokens.
  • Authorization: Bearer 09ea8435-403d-4062-bd84-a6907b01c9ef
Error Handling -> Parse the JSON response 'status' field will have the value of '0', this means the 'error' field will have the error details.
status: 0 ,
error:
   errorCode: "PN01",
   message: "Dude you are trying to update a non-existing account. Please stop drugs."

JSON Response -> API response will be on json format. If 'status' field is 1 this means the requests is successful, response data will be on the 'data' field.

    status: 1,
    data:
      "fullName": "chi long qua",
      "deviceId": "666"
    
    error:
      errorCode: null,
      message: null,
